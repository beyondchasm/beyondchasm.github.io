# 투둥 개인정보 처리방침

- 시행일: 2026-03-10
- 최종 수정일: 2026-03-10

투둥(이하 "앱")은 사용자의 할 일 관리 경험을 제공하기 위해 필요한 최소한의 정보만 처리합니다. 본 개인정보 처리방침은 앱이 어떤 정보를 어떻게 처리하는지 설명합니다.

## 1. 수집 및 저장하는 정보

앱은 다음 정보를 기기 내부에 저장할 수 있습니다.

- 할 일 제목, 메모, 마감일시, 분류 정보, 완료 여부
- 완료 기록
- 앱 설정값
  - 테마
  - 폰트
  - 정렬 및 보기 설정
  - 분류 보드 이름 및 사용 여부
- 알림 설정값
  - 하루 요약 사용 여부와 시간
  - 마감 알림 사용 여부와 기본 시간
- 보안 설정값
  - PIN 잠금 사용 여부
  - PIN 인증에 필요한 기기 내부 보안 정보

## 2. 개인정보의 수집 방법

앱은 회원가입이나 로그인 기능을 제공하지 않으며, 사용자가 앱 화면에 직접 입력한 정보만 처리합니다.

다음 경우에 정보가 생성되거나 저장될 수 있습니다.

- 사용자가 할 일을 작성하거나 수정하는 경우
- 사용자가 완료 처리, 복구, 삭제를 수행하는 경우
- 사용자가 설정을 변경하는 경우
- 사용자가 백업 또는 복구 기능을 사용하는 경우

## 3. 정보 저장 위치

앱의 주요 데이터는 기본적으로 사용자의 기기 내부 저장소 및 로컬 데이터베이스(SQLite)에 저장됩니다.

현재 버전의 앱은 다음 기능을 제공하지 않습니다.

- 계정 기반 로그인
- 개발사 서버로의 자동 업로드
- 클라우드 동기화

## 4. 외부 전송 및 제3자 제공

앱은 원칙적으로 사용자의 할 일 데이터와 설정 데이터를 개발사 서버로 전송하거나 제3자에게 판매 또는 제공하지 않습니다.

다만 다음 경우에는 사용자의 선택에 따라 외부 저장이 발생할 수 있습니다.

- 사용자가 백업 파일을 직접 생성하여 기기 파일 시스템 또는 외부 저장 위치에 저장하는 경우
- 사용자가 기기 알림 기능을 사용하기 위해 운영체제 권한을 허용하는 경우
- 사용자가 정책 문서 또는 외부 링크를 열기 위해 브라우저를 실행하는 경우

## 5. 알림 권한 사용

앱은 다음 목적에 한하여 알림 권한을 사용할 수 있습니다.

- 하루 요약 알림 발송
- 할 일의 마감일 또는 마감 시각에 따른 알림 발송

앱은 광고성 알림을 발송하지 않습니다.

## 6. 보안

앱은 사용자의 민감한 생산성 데이터를 보호하기 위해 합리적인 수준의 보호 조치를 적용합니다. 다만, 기기 자체의 분실, 탈옥 또는 루팅, 운영체제 보안 약화 등 사용자의 기기 환경으로 인한 위험까지 완전히 보장할 수는 없습니다.

사용자가 PIN 잠금 기능을 활성화한 경우, 앱은 기기 내부에 해당 보안 설정 정보를 저장하여 잠금 기능을 제공합니다.

## 7. 백업 및 복구

앱은 사용자가 직접 백업 파일을 생성하고 복구할 수 있는 기능을 제공할 수 있습니다.

- 백업 파일에는 사용자의 할 일 데이터, 완료 기록, 일부 설정 정보가 포함될 수 있습니다.
- 보안상 이유로 PIN 자체 또는 복구 코드 자체는 백업 파일에 포함되지 않을 수 있습니다.
- 사용자는 백업 파일을 안전한 위치에 직접 보관할 책임이 있습니다.

## 8. 개인정보 보유 기간

앱은 사용자가 삭제하지 않는 한, 입력된 데이터와 설정을 사용자의 기기 내부에 보관합니다.

다음 경우 관련 정보는 삭제될 수 있습니다.

- 사용자가 개별 할 일 또는 완료 기록을 삭제한 경우
- 사용자가 전체 데이터 초기화를 실행한 경우
- 사용자가 앱을 삭제하고 관련 로컬 데이터가 운영체제에 의해 제거된 경우

## 9. 사용자의 권리

사용자는 언제든지 다음을 수행할 수 있습니다.

- 자신의 할 일 및 완료 기록을 열람, 수정, 삭제
- 설정값 변경
- 백업 파일 생성 및 삭제
- 전체 데이터 초기화
- 앱 삭제를 통한 로컬 데이터 제거

## 10. 아동의 개인정보

앱은 아동을 대상으로 별도의 회원가입 또는 개인정보 제출 절차를 운영하지 않습니다. 앱은 사용자가 직접 입력한 정보만 기기 내부에 저장합니다.

## 11. 정책 변경

본 개인정보 처리방침은 앱 기능 변경, 법령 변경, 운영 정책 변경에 따라 수정될 수 있습니다. 중요한 변경이 있는 경우 앱 내 고지, 업데이트 노트, 또는 연결된 정책 페이지를 통해 안내할 수 있습니다.
