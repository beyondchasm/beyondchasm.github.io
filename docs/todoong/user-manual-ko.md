# 투둥 사용자 메뉴얼

- 문서 버전: 1.0
- 작성일: 2026-03-10

이 문서는 투둥 앱을 처음 사용하는 사용자가 주요 기능을 빠르게 이해하고 활용할 수 있도록 정리한 안내 문서입니다.

## 1. 투둥은 어떤 앱인가요?

투둥은 해야 할 일을 빠르게 적고, 분류 보드로 정리하고, 완료 기록까지 관리할 수 있는 개인용 할 일 앱입니다.

투둥은 다음 흐름에 맞춰 설계되어 있습니다.

1. 해야 할 일을 빠르게 적습니다.
2. 필요하면 분류와 마감일시를 정리합니다.
3. 진행 중인 일은 홈에서 관리합니다.
4. 끝낸 일은 완료 기록에서 다시 확인합니다.
5. 알림, 테마, 백업, 잠금 같은 설정은 설정 화면에서 관리합니다.

## 2. 화면 구성

### 홈

홈은 진행 중인 할 일을 관리하는 화면입니다.

- 할 일 추가
- 할 일 수정
- 분류 이동
- 완료 처리
- 검색
- 여러 항목 선택 관리

### 완료 기록

완료 기록은 끝낸 할 일을 다시 보는 화면입니다.

- 기간별 보기
- 검색
- 정렬
- 복원
- 삭제

### 설정

설정에서는 앱의 동작 방식과 표시 방식을 관리합니다.

- 알림
- 테마와 폰트
- 보기와 정리
- 분류 보드
- 보안
- 백업과 데이터
- 정책 및 앱 정보

## 3. 시작하기

투둥을 처음 쓸 때는 아래 순서로 시작하는 것을 권장합니다.

1. 홈에서 `+` 버튼을 눌러 할 일을 추가합니다.
2. 바로 분류하기 어렵다면 먼저 `미분류`에 둡니다.
3. 중요한 할 일에는 마감일 또는 마감 시각을 설정합니다.
4. 설정에서 필요한 경우 알림을 켭니다.
5. 자주 쓰는 테마, 폰트, 분류 이름을 설정합니다.

## 4. 홈 화면 사용법

### 4-1. 할 일 추가하기

홈 오른쪽 아래 `+` 버튼을 누르면 할 일을 추가할 수 있습니다.

기본적으로 다음 정보를 입력할 수 있습니다.

- 제목
- 메모
- 분류
- 마감일
- 마감 시각

제목만 입력해도 저장할 수 있으며, 세부 정보는 나중에 다시 수정할 수 있습니다.

### 4-2. 마감일과 시간 설정하기

마감일 입력 화면에서는 날짜와 시간을 한 흐름 안에서 설정할 수 있습니다.

- 날짜만 설정할 수 있습니다.
- 날짜와 시간을 함께 설정할 수 있습니다.
- 시간이 필요 없으면 해제할 수 있습니다.

마감 시각이 설정된 할 일은 마감 알림 기준에도 반영됩니다.

### 4-3. 할 일 수정하기

할 일을 열면 제목, 메모, 분류, 마감일시를 다시 수정할 수 있습니다.

내용이 길거나 맥락이 필요한 일은 메모를 함께 적어 두는 것이 좋습니다.

### 4-4. 완료 처리하기

홈에서 진행 중인 할 일은 탭으로 완료 상태를 바꿀 수 있습니다.

완료된 항목은 즉시 사라지지 않을 수 있으며, 정리 흐름에 따라 완료 기록으로 이동해 보관할 수 있습니다.

### 4-5. 드래그로 분류 바꾸기

홈의 할 일은 길게 눌러 드래그한 뒤 다른 분류 칸으로 옮길 수 있습니다.

이 방식은 빠르게 분류를 정리할 때 가장 편리합니다.

### 4-6. 검색하기

홈 상단의 검색 버튼을 누르면 검색 오버레이가 열립니다.

검색은 다음 내용을 기준으로 동작합니다.

- 할 일 제목
- 메모

생각나는 단어 일부만 입력해도 관련 항목을 빠르게 좁혀 볼 수 있습니다.

### 4-7. 여러 항목 선택하기

홈 상단의 선택 버튼을 누르면 선택 모드가 켜집니다.

선택 모드에서는 여러 할 일을 한 번에 고른 뒤 다음 작업을 할 수 있습니다.

- 완료
- 분류 이동
- 삭제

선택 모드는 상단의 `X` 버튼으로 종료할 수 있습니다.

## 5. 분류 보드 이해하기

투둥은 기본적으로 다섯 영역으로 일을 나눠 볼 수 있습니다.

### 미분류

아직 어디에 둘지 정하지 않은 일을 모아두는 기본 영역입니다.

### 분류 1

가장 먼저 처리해야 하는 중요한 일을 두는 영역입니다.

### 분류 2

미리 준비하면 좋은 중요한 일을 두는 영역입니다.

### 분류 3

빠르게 정리하거나 짧게 대응할 일을 두는 영역입니다.

### 분류 4

당장 중요하지 않거나 나중에 봐도 되는 일을 두는 영역입니다.

분류 이름은 설정의 `분류 보드`에서 원하는 이름으로 바꿀 수 있습니다.

## 6. 완료 기록 사용법

완료 기록은 이미 끝낸 일을 다시 확인하는 공간입니다.

### 6-1. 완료 기록 보기

완료 기록 화면에서는 완료한 할 일을 날짜 기준으로 모아 볼 수 있습니다.

연도와 월을 기준으로 원하는 기간만 좁혀 볼 수 있습니다.

### 6-2. 검색과 정렬

완료 기록 상단의 액션 버튼으로 다음 기능을 사용할 수 있습니다.

- 검색
- 정렬
- 선택

검색은 완료된 할 일의 제목과 메모를 기준으로 동작합니다.

### 6-3. 여러 항목 관리하기

완료 기록 상단의 선택 버튼을 누르면 선택 모드가 켜집니다.

선택한 기록은 한 번에 다음 작업을 할 수 있습니다.

- 복원
- 삭제

## 7. 알림 사용법

알림 설정은 `설정 > 알림`에서 관리합니다.

### 7-1. 하루 요약

하루 요약은 하루에 한 번, 오늘의 일정 흐름을 점검할 수 있도록 알려주는 알림입니다.

사용자는 다음을 설정할 수 있습니다.

- 사용 여부
- 보낼 시간

### 7-2. 마감 알림

마감 알림은 할 일의 마감일 또는 마감 시각에 맞춰 알려주는 알림입니다.

현재 동작 기준은 다음과 같습니다.

- 마감 시각이 있으면 그 시각에 알림
- 시간이 없고 날짜만 있으면 `기본 시간`에 알림

사용자는 다음을 설정할 수 있습니다.

- 사용 여부
- 기본 시간

알림 설정을 바꾼 뒤에는 반드시 저장 버튼을 눌러야 반영됩니다.

## 8. 테마와 폰트

`설정 > 테마`에서 앱의 인상을 바꿀 수 있습니다.

### 8-1. 테마

앱은 밝기 모드와 색상 스타일을 설정할 수 있습니다.

- 시스템
- 라이트
- 다크

### 8-2. 폰트

사용자는 앱 전체 폰트를 선택할 수 있습니다.

- 시스템 폰트
- 앱에서 제공하는 폰트 프리셋

폰트를 바꾸면 홈, 완료 기록, 설정 등 앱 전반의 타이포그래피에 반영됩니다.

## 9. 보기와 정리

`설정 > 보기와 정리`에서는 목록이 보이는 방식과 기본 정리 기준을 바꿀 수 있습니다.

예를 들어 다음 항목을 조정할 수 있습니다.

- 홈 목록 정렬
- 완료 기록 정렬
- 마감일 표시 방식
- 마감일 라벨 형식

## 10. 분류 보드 설정

`설정 > 분류 보드`에서는 분류 이름과 사용 여부를 관리할 수 있습니다.

### 10-1. 이름 바꾸기

목록에서 원하는 행을 누르면 이름 수정 시트가 열립니다.

여기서 분류 이름을 더 익숙한 표현으로 바꿀 수 있습니다.

예시:

- 분류 1 → 오늘 꼭 할 일
- 분류 2 → 미리 준비
- 분류 3 → 빠른 처리
- 분류 4 → 나중에 검토

### 10-2. 분류 끄기

선택 분류는 오른쪽 토글로 끄거나 켤 수 있습니다.

어떤 분류를 끌 때 그 안에 진행 중인 할 일이 있으면, 해당 항목은 미분류로 이동한 뒤 저장됩니다.

## 11. 보안 기능

`설정 > 보안`에서는 앱 잠금 기능을 설정할 수 있습니다.

사용 가능한 기능은 다음과 같습니다.

- PIN 잠금 설정
- PIN 변경
- 복구 코드 관련 기능

앱 잠금을 켜면 앱 진입 시 PIN 입력 화면이 표시될 수 있습니다.

## 12. 백업과 데이터

`설정 > 백업과 데이터`에서는 백업, 복구, 데이터 정리 기능을 사용할 수 있습니다.

### 12-1. 백업

현재 상태를 백업 파일로 저장할 수 있습니다.

백업에는 다음 정보가 포함될 수 있습니다.

- 할 일 데이터
- 완료 기록
- 앱 설정
- 알림 설정

보안상 이유로 PIN 자체는 백업에 포함되지 않을 수 있습니다.

### 12-2. 복구

저장된 백업 파일을 기준으로 데이터를 복구할 수 있습니다.

복구 전에는 어떤 데이터가 들어 있는지 미리 확인하는 것이 좋습니다.

### 12-3. 완료 기록 비우기

완료 기록만 따로 비울 수 있습니다.

이 작업은 진행 중인 할 일에는 영향을 주지 않습니다.

### 12-4. 전체 데이터 초기화

전체 데이터 초기화를 실행하면 다음 정보가 함께 초기화될 수 있습니다.

- 진행 중인 할 일
- 완료 기록
- 앱 설정
- 알림 설정
- PIN 잠금 상태

실행 전에는 백업을 먼저 만드는 것을 권장합니다.

## 13. 개인정보와 저장 방식

투둥은 현재 계정 로그인이나 서버 동기화를 사용하지 않습니다.

주요 데이터는 기본적으로 기기 내부 저장소에 보관됩니다. 사용자가 직접 실행한 백업 또는 외부 링크 열기 같은 경우를 제외하면, 사용자의 할 일 데이터를 외부 서버로 자동 전송하지 않습니다.

자세한 내용은 개인정보 처리방침 문서를 확인해 주세요.

## 14. 자주 묻는 질문

### Q. 완료한 일이 왜 바로 없어지지 않나요?

투둥은 완료한 일을 바로 버리기보다, 완료 기록 흐름 안에서 다시 확인할 수 있게 설계되어 있습니다.

### Q. 알림이 오지 않아요.

다음을 순서대로 확인해 주세요.

1. 운영체제 알림 권한이 켜져 있는지
2. 앱 안에서 해당 알림이 `사용`으로 설정돼 있는지
3. 설정 변경 후 저장 버튼을 눌렀는지
4. 기기 절전 모드나 제조사 알림 제한이 있는지

### Q. 분류 이름을 꼭 기본값으로 써야 하나요?

아닙니다. 분류 보드에서 자유롭게 바꿀 수 있습니다.

### Q. 기기를 바꾸기 전에 무엇을 하면 좋나요?

백업을 먼저 만든 뒤, 새 기기에서 복구하는 것을 권장합니다.

## 15. 이런 식으로 쓰면 편합니다

### 먼저 적고 나중에 분류하기

생각나는 할 일을 즉시 적고, 분류는 나중에 한꺼번에 정리하는 방식이 오래 가기 쉽습니다.

### 마감 시각은 꼭 필요한 일에만 넣기

모든 일에 시간을 넣기보다, 실제로 시각이 중요한 일정에만 시간을 주면 알림 피로가 줄어듭니다.

### 큰 변경 전에는 백업 먼저 하기

분류를 크게 바꾸거나 데이터를 정리하기 전에는 백업을 먼저 만들어 두는 것이 안전합니다.
